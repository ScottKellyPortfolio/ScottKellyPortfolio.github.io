Projects

The Breakes - Band Website

Design, Development

What started as a basic, static HTML/CSS/JS website for my brothers band has turned into my largest single personal project. As I mentioned, this website is for my brothers band, The Breakes. I created the site originally as a static website as a simple project when I was still very green in the world of coding. I worked with the band to design a fairly basic informational site and this is the result. Though it looks similar to the first iteration, it has evolved into something far more.

As it was originally built as a static site, it meant that when anything needed changed, such as adding a show event or pictures or other info, I had to go and manually modify the HTML. This wasn't exactly maintainable, so I set out working on the second iteration of the site. What I ended up creating was something that resembles a basic CMS (Content Management System). While a very basic CMS, it allowed the guys, all 3 of them, access and the ability to add and manage all of their own content. This meant no more coming to me for modifying HTML. The site is built using ASP.NET MVC. It uses Entity Framework and a code-first pattern for all data transactions. Most of the HTML/CSS/JS was carried over, and modified to work with dynamic rendering with MVC. This brought along with it a few hurdles. While I was vaguely familiar with hosting a website, I hadn't full on hosted my own website. I had to figure out what to do for a hosting provider, DNS hosting, a database, the whole nine-yards. Though a struggle for a while, and more than one long night, it was greatly helpful in helping me understand how a website fundamentally works.

The latest iteration was sort-of built on top of what was there, but not really. For my latest challenge, I decided to rebuild the site using .NET Core. While that may not sound very different from the framework I used before, functionally, it was quite different. This being my first proper attempt at a project using .NET Core, I decided to go all in. I re-wrote the views (HTML pages) to use more modern tag-helpers, started using Razor Pages, View Components, and still combined regular MVC patterns. There's a dab of React in there as well ;) I reworked what I needed to change to make authentication work with .NET Core, but I also created a full REST API with full CRUD operations for the miscellaneous aspects of the 'CMS'. This REST API is what I would use for my next major project; The Breakes Manager, a mobile app to manager the website content.

The Challenge

Provide a modern experience, for both site consumers and content admins. This means a responsive website, with features for admins to management content with little assistance from IT/Engineering (me).

The Solution

A website that allows content admins the ability to maintain, modify and manage all content aspects of the website. The current iteration is built on a foundation .NET Core, with a myraid of other technologies.

The Result

Check it out here TheBreakes.com. I'm pretty proud of this site, particularly the content management side of things. I've got more things planned for the future, so this won't be the final iteration.

The Breakes Manager

Design, Development

I know, it's not exactly a creative name up front, but if you think about it, it's kinda like having a manager for the band, just in digital form! Right?! No? Well either way, name aside, this is the mobile app I developed for The Breakes website. Assuming you've made it this far after having read the project notes about The Breakes band website, you may know I briefly talked about a mobile app. This app was a self proposed challenge as a way to get into the world of mobile app development. The purpose of this site is to provide a mobile management experience for the band. While it may be a bit overkill, as the main website backend is responsive and can be used on a phone, this is a more streamlined process.

Being a primarliy Microsoft technologies guy, I chose the obvious place to start with this; Xamarin.Forms. I set out building this using the REST API I developed during the latest iteration of the website build. The website uses a customized, scaffolded, Identity Server pattern for authenticating users and OAuth for the workflow. This meant that I had to adopt that with this mobile app to gain access to the API. I took an approach of caching a users login information to their device to avoid having to login everytim they open the app. Within the app, they have all of the standard management options as the website. This has the advantage for uploading different types of media such as videos and images, as they can upload directly from their phones. The app is not entirely finished, and I envision it going on for a while, but it's been a good experience.

The Challenge

Self proposed, this was a personal goal I'd wanted to tackle for a while. Learning a new platform and many new patterns was quite a daunting task.

Weight And Balance Calculator

Design, Development

The idea for this application came from my father, who pilots private charter jets. The purpose of the app, is that it makes it simple to calculate the weight and balance of an aircraft before it flies. It uses built in information for different aircraft to make it simple to give your weights as inputs, and it will tell you if the aircraft is safe to fly for the weights the user input. At this point in time, it is still a work in progress, but is close to being finished.

The Challenge

Simplify calculating the various weights of small aircraft to make calculating the weight and balance easier. Make it easy to use and make it mobile friendly so as to avoid needing a desktop or laptop.

The Solution

A responsive web application that can be used on mobile devices to increase ease and productivity. Replaces the need to use a computer or desktop application.

The Result

Still to come! This is a work in progress project as I am the only person working on it. Unfortunately because it is a personal project, it takes the backseat for more pressing issues.
